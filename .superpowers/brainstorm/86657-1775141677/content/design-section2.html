Key design rules:
• Both card types follow the same pattern: collapsed = fit-content pill, expanded = wider card
• Collapsed shows only: status icon + label (e.g. "Web Search" or "Thought (2.3s)") + chevron
• Expanded shows the same header row + content area below
• Tool call content: rendered in a monospace code block with line wrapping
• Thinking content: rendered as markdown text (existing behavior)
• Running state: shows spinner icon, never expandable (content not ready yet)
• Default state: always collapsed when completed
